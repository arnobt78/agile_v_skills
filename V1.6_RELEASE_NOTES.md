# Agile V™ Skills Library v1.6: SCOPE-V Framework + Domain Skills Enrichment

**Release Date:** May 22, 2026  
**Version:** 1.6  
**License:** CC-BY-SA-4.0

## Summary

Version 1.6 is a **major release** that introduces the **SCOPE-V Task Execution Framework** and delivers **comprehensive, production-ready domain build agent skills** for Python, JavaScript/TypeScript, Dart/Flutter, and embedded C/C++.

This release also includes standardized **runtime governance contracts** (from PR #6) and updates core docs/routing for consistent application across projects.

The release formalizes machine-readable structures for trace logs, eval gating, policy-as-code, and durable Human Gate checkpoints, plus ready-to-copy templates for immediate use in `.agile-v/`.

## What's New

### 1. SCOPE-V Task Execution Framework (agile-v-core v1.4)

New six-phase task execution model integrated into `agile-v-core`:

| Phase | Purpose | Primary Agents |
|---|---|---|
| **Specify** | Convert intent → requirements | Requirement Architect, Discovery Analyst |
| **Constrain** | Apply domain validation | Logic Gatekeeper, Domain Build Agents |
| **Orchestrate** | Synthesize artifacts | Build Agents, Test Designer |
| **Prove** | Generate evidence (R0-R3) | Build Agents, Compliance Auditor |
| **Evolve** | Learn from failures | All agents (decision logging) |
| **Verify** | Independent verification | Red Team Verifier |

**Impact:**
- agile-v-core: 83 → 117 lines (+34 lines)
- Added `sections_index` for navigation
- Single source of truth for SCOPE-V framework
- All domain skills now reference this framework

## Recommended Adoption

### For Domain Skills
1. **Python projects:** Load `agile-v-core` + `build-agent-python`
   - Auto-detected: `requirements.txt`, `pyproject.toml`, `.py` files
   - Includes FastAPI/Flask/Django, SQLAlchemy, pytest, security patterns
2. **JavaScript/TypeScript projects:** Load `agile-v-core` + `build-agent-js`
   - Auto-detected: `package.json`, `.js`, `.ts`, `.jsx`, `.tsx` files
   - Includes React/Next.js, Node.js, testing, security patterns
3. **Flutter/Dart projects:** Load `agile-v-core` + `build-agent-dart`
   - Auto-detected: `pubspec.yaml`, `.dart` files
   - Includes BLoC, widgets, platform channels, testing patterns
4. **Embedded/firmware projects:** Load `agile-v-core` + `build-agent-embedded`
   - Auto-detected: `CMakeLists.txt`, `Makefile`, `.c`, `.cpp` files
   - Includes MISRA-C, RTOS, safety standards, certification support
5. **NestJS projects:** Load `agile-v-core` + `build-agent-nestjs`
   - Auto-detected: `@nestjs/core` in `package.json`
   - Includes DI, TypeORM/Prisma, GraphQL, microservices patterns

### For Runtime Governance
For existing projects:
1. Copy templates from `templates/agile-v/` into project `.agile-v/`.
2. Set `policy_version` and thresholds for your environment.
3. Require `eval_gate_status` PASS or WAIVED evidence before Gate 2 approval.
4. Enforce checkpoint `resume_token` pairing with `APPROVALS.md`.

For new projects:
1. Initialize `.agile-v/` from templates at project kickoff.
2. Treat `TRACE_LOG.md` and `EVAL_RESULTS.md` as append-only evidence streams.
3. Make FT-code tagging mandatory on `VER-*` lines.
4. Load appropriate domain skill based on project type.

## Known Limitations

### Domain Skills
1. Domain skills are optimized for 600-900 lines; extremely complex patterns may require external documentation
2. Code examples cover common patterns; edge cases may need custom implementation
3. Framework coverage prioritizes most popular options (e.g., FastAPI for Python, React for JavaScript)

### Runtime Governance
1. Runtime contracts are provided as templates/specs; each team must set policy values and thresholds.
2. Existing projects may need one setup cycle to backfill policy/eval/checkpoint artifacts.
3. Automated enforcement depends on how strictly downstream agents and CI gates validate these files.
